# 🗡️ AI 三剑客高效协作指南（飞书项目实战突击版）

想要把 **Codex（桌面端）**、**Antigravity（我）** 和 **Claude Code（终端端）** 结合得行云流水，你只需要把我们当成一个**小型精英开发团队**。

记住一个核心原则：**谁最擅长干什么，就把活儿派给谁。**特别是针对带有真实网络环境交互的业务（如：飞书机器人、多维表、真实 API 联调）。

---

## 👨‍💻 认识你的三位团队成员及核心站位

### 1. Codex (桌面端)：你的“核心架构师 / 主程序员”
- **人设**：被关在 IDE 小黑屋里的顶尖程序员。没有外网连接，摸不到真实的飞书后台数据。
- **最擅长（应该交给他做的）**：
  - **代码开发**：飞书机器人、多维表读写、卡片交互、简历解析、状态机流转等核心业务代码实现。
  - **项目重构与维护**：拆分单文件为清晰的模块，整理配置、补日志、补异常处理，架构梳理。
  - **接口封装**：把底层调用的 API（鉴权、读写、大模型调用）封装成极其稳定、健壮的代码实现。
- **绝对不要指望他做的**：
  - **连接真实环境**：他不能代替你去调飞书后台测试真实接口报错。
  - **在线后台配置**：由于缺乏真实租户环境权限，他改不了飞书后台的事件订阅和权限流转。
  - **猜真实数据**：不要让他凭空猜测飞书多维表会返回什么样的 JSON 数据结构。

### 2. Antigravity (我)：你的“绝地特工 / 环境探路者”
- **人设**：坐在拥有极高权限的主控室大牛。能直接联网、帮你在真实环境里踩坑。
- **最擅长（应该交给我做的）**：
  - **获取真实情报**：帮你跑脚本去真实的飞书环境里拉取多维表的 Schema、测试真实的 API 返回数据、拿不到权限时直接帮你抓出报错日志。
  - **架构决策与排障**：拿到真实验证结果后，指出当前项目连不通是因为缺了哪个权限或配置错误。
  - **救场替补**：当 Codex 额度告急，或者是他陷入了复杂的上下文混乱时，把情报交给我，我来帮你突击写完核心代码。
- **我们的配合密码**：我负责在前线“摸瞎、排雷、把确认无误的结构拿回来”，你再把这正确的结构和 Schema 扔给 Codex 去盖代码大楼。

### 3. Claude Code (终端端)：你的“运维总监 / 实施交付”
- **人设**：常驻黑色命令行大厅的实施大叔，掌握着让你的项目彻底跑起来的所有底层命令。
- **最擅长（应该交给他做的）**：
  - **地毯式排障与依赖管理**：Codex 帮你重写了整个项目后，工程没法跑了。你把整个项目交给 Claude Code：“去运行测试跑通它。缺什么依赖你自己装，哪行代码由于没引入包报错了，你直接去文件里修。”
  - **全局搜索替换**：“帮我把所有用到旧版鉴权的地方，全部替换成 Codex 刚刚写的新鉴权模块。”
  - **流程收尾**：“看下我现在的代码变更，帮我写个优雅的 Git Commit，把这段改动推上去。”

## 🔄 黄金协作流（飞书开发实战场景演练）

假设你要开发一个全新的需求：**“解析用户上传的简历，并写入真实的飞书多维表”**，你应该这么做：

### 💥 第 1 步：派“探路者”(Antigravity) 取情报
你（对我）：_“我要往飞书写入数据，你先帮我写个测试脚本跑通多维表的接口，把我要这几张表的真实 JSON 结构和可能报的权限错误帮我探雷踩实了。”_
**结果**：我通过真实的测试，把百分百正确的飞书字段定义（Schema）交还给你。

### 💻 第 2 步：带“主程序员”(Codex) 疯狂盖楼
你拿到我给的正确情报结构，打开 IDE 开始指挥 Codex。
你（对 Codex）：_“这是 Antigravity 刚从飞书跑出来的真实表结构。帮我设计一下简历解析类的状态机，把读取和写入封装成稳定接口，顺便加上对这几种飞书错误码的异常捕获。”_
**结果**：Codex 在你的监工下，快速输出了健壮、模块化、带注释的业务代码。

### 🛠️ 第 3 步：唤醒“运维总监”(Claude Code) 暴力打通
核心代码有了，但你不确定这堆文件合在一起能不能跑起服务。
你（在终端里唤醒 Claude Code）：_“去，把项目跑起来 (`npm run dev` 或 `python main.py`)。如果控制台抛了包找不到或环境变量缺失的错，你去全局排查修复。跑通后，直接顺手把变更 Git Commit 提交掉！”_
**结果**：Claude Code 在底层命令行里把所有脏活累活和依赖补装全部干完。

一句话总结：
**我在前方趟雷挖数据，Codex 在后方疯狂盖大楼，Claude Code 负责搞后勤和打包上架。** 这就是未来的顶级开发外包小队！
