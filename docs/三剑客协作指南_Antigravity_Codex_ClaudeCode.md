# AI 三剑客项目协作指南 (飞书开发专项)

本文档旨在明确 Codex (桌面端)、Antigravity (网页/对话端) 和 Claude Code (终端端) 三个 AI 工具在实际开发中的分工与协作流程，特别是针对涉及真实环境交互（如飞书 API 调用、多维表读写等）的复杂项目。

核心协作原则：基于各自的运行环境和权限限制，分配最适合的任务。

---

## 一、 各工具定位与分工

### 1. Codex (桌面代码助理)

**适用场景**：日常编码、架构设计、逻辑实现。
**运行环境**：IDE/编辑器内部（无外部真实网络环境的直接操作权限）。

**擅长（应交办的任务）**：
- **核心代码编写**：根据确定的 JSON 结构或 Schema，编写飞书机器人状态机、多维表读写逻辑、卡片交互解析等业务代码。
- **项目模块化重构**：将臃肿的单文件脚本拆分为结构清晰的模块，补充日志和异常处理逻辑，优化可维护性。
- **接口封装**：将底层功能（如飞书鉴权、API 调用、大模型交互）抽象并封装为稳定的函数或类。
- **架构设计**：梳理业务流程，设计字段模型和数据结构。

**不适合的工作（避免交办的任务）**：
- 替你执行真实的 API 请求来验证连通性。
- 排查只有在真实报错反馈中才能确认的权限、回调失败等问题。
- 在缺乏官方文档或真实返回值参考的情况下，凭空猜测飞书接口的返回结构。

### 2. Antigravity (智能协助引擎)

**适用场景**：真实环境测试、复杂问题排查、架构确认、方案预研。
**运行环境**：拥有较高权限的运行环境（可联网、可执行脚本）。

**擅长（应交办的任务）**：
- **环境交互与数据获取**：代替 Codex 连接真实的飞书环境，读取真实的多维表结构、调用 API 以获取真实的 JSON 返回值，以消除对数据结构的猜测。
- **权限与环境排错**：通过在线联调排查报错原因（如网络受限、Tenant Access Token 过期、特定字段权限缺失等）。
- **兜底与复杂技术攻关**：在 Codex 额度耗尽，或面临特别复杂难以一次性生成的逻辑块时，提供代码实现支持。

### 3. Claude Code (终端执行代理)

**适用场景**：工程环境管理、批量化操作、项目级调试。
**运行环境**：系统的命令行终端（拥有文件系统全权限和命令行执行权）。

**擅长（应交办的任务）**：
- **环境依赖配置**：快速运行 `npm install` 或 `pip install`，根据错误提示自动排查并补全缺失的包。
- **工程级搜索与替换**：在大型项目中执行全局查找和替换操作（如统一修改某旧版 API 的调用方式）。
- **运行调试与收尾**：执行服务的启动脚本或测试脚本，根据终端报错日志定位并修复运行时的环境错误。协助整理代码变更并生成 Git 提交记录。

## 二、 典型操作工作流（以飞书多维表写入需求为例）

**1. 信息采集与环境验证 (负责: Antigravity)**
- **操作指令示例**：“帮我写个 Python 测试脚本，调用该多维表的开放接口，返回真实的字段结构 (Schema)，并确认当前的 API Token 是否有对应的写入权限。”
- **产出**：获取经过真实环境验证的 JSON 数据结构和权限状态。

**2. 核心架构设计与编码 (负责: Codex)**
- **操作指令示例**：“这是 Antigravity 返回的真实飞书多维表 Schema。请根据这个结构，使用对应的 SDK 帮我封装一个稳定写入多维表的方法，并补充针对 Token 失效和网络超时的异常捕获逻辑。”
- **产出**：输出结构严谨、可直接集成的业务代码，并能在 IDE 中通过代码建议进行快速补全。

**3. 项目编译与运行排错 (负责: Claude Code)**
- **操作指令示例**：“在终端尝试运行当前项目。如果发现缺失依赖，请自动安装；如果运行存在报错，请根据日志修复代码。确认一切正常后，帮我提交 git commit。”
- **产出**：解决环境依赖问题，确保项目正常运行并完成版本控制收尾。

---

## 三、 协作总结

- **Antigravity** 负责**与真实环境交互**，确认 API 返回的正确信息并排除权限/网络类干扰。
- **Codex** 拿着确认好的确切信息，在 IDE 内进行**设计架构和沉浸式编码**。
- **Claude Code** 负责**环境配置和工程级别的自动化调试/执行**。
